## STEP 1 — X（Twitter）自動投稿【完全自動・最優先】

### 何をするか
毎朝、yfinance でVIXと日本株データを取得 → Claude で日本語ツイートを生成 → X API で自動投稿。
既存の `market_data.py` をほぼそのまま流用できる。

### 投稿内容（自動生成）
- 毎朝8時（市場開幕前）: VIXレベル + 本日の相場コンディション一言コメント
- 毎夕17時（市場終了後）: 日経平均の動き + 翌日の注目点
- 週1回: 「先週AIが提案した銘柄の結果」（Nightly Edgeの実績報告）

### 実装手順

**1-1. Twitter Developer Portalでアプリ作成**
1. https://developer.twitter.com → 無料アカウント作成
2. App作成 → OAuth 2.0 を有効化
3. 以下のキーを取得してメモ：
   - API Key / API Secret
   - Access Token / Access Token Secret
4. 無料枠: 月1,500ツイートまで書き込み可（1日50投稿で十分）

**1-2. 自動投稿スクリプト作成**

`scripts/post_market_tweet.py` を作成して以下のロジックを実装：

```python
# 擬似コード（実装時はここに書く）
1. yfinance で VIX・日経・主要銘柄を取得
2. Claude API でツイート文（140字以内）を生成
3. tweepy で X に投稿
```

使用ライブラリ: `tweepy`（Twitter API公式ラッパー）

**1-3. GitHub Actions でスケジュール実行**

`.github/workflows/tweet.yml` を作成：

```yaml
on:
  schedule:
    - cron: '0 23 * * 0-4'   # 日本時間 朝8時（UTC 23:00）月〜金
    - cron: '0 8 * * 1-5'    # 日本時間 夕17時（UTC 8:00）月〜金
```

GitHub Actions は無料枠（2,000分/月）で十分動く。
シークレットキーは GitHub → Settings → Secrets に登録。

**1-4. 投稿する内容の方向性**
- `#日本株` `#個人投資家` `#VIX` タグを固定
- 「今日のVIX: 18.3（安全圏）日本株は落ち着いた動き。Nightly Edgeで翌日計画を立てましょう」のような型
- Nightly EdgeのURLをプロフィールに固定（ツイートごとには入れない。スパム扱い回避）

## STEP 2 — ニュースレター（Beehiiv）【週1回確認のみ】

### 何をするか
週次の日本株市場レポートをClaudeが生成 → 人間が5分確認・推敲 → Beehiivから送信。
将来的に有料読者からの収益化も可能。

### 設定手順

**2-1. Beehiiv アカウント作成**
1. https://www.beehiiv.com → 無料プラン（2,500読者まで無料）
2. パブリケーション名: 「Nightly Edge Weekly」
3. 購読フォームのURLを取得 → Nightly EdgeのLPと設定ページに埋め込む

**2-2. ニュースレター内容テンプレ（Claude が毎週生成）**

```
件名: 【週次】今週の日本株まとめと来週の注目点

■ 今週のVIX推移と相場コンディション
■ 今週の主要指数パフォーマンス
■ 来週の注目マクロイベント（決算・指標）
■ Nightly Edgeユーザー向けヒント
■ 今週の用語解説（初心者向け）
```

**2-3. 週次生成スクリプト**
GitHub Actions で毎週日曜夜に実行 → Markdownファイルをリポジトリに保存 → 人間が確認してBeehiivに貼り付け → 送信（5分作業）

**2-4. 収益化タイミング**
- 読者500人超: Beehiivの「Recommendations」機能でマネタイズ開始（他ニュースレターを紹介して報酬）
- 読者1,000人超: 有料プラン（$5〜$10/月）導入を検討

## STEP 3 — note 有料記事【人間が推敲、投稿は手動】

### 何をするか
Claude が教育的な投資記事を生成 → 人間が30分推敲 → noteで100〜300円の有料記事として販売。
一度書いたら永続的に売れ続ける。

### 記事テーマ（Claude が下書き生成、人間がレビュー）

優先度高（検索需要あり）：
1. 「VIXとは何か？個人投資家が知るべき恐怖指数の使い方」
2. 「ATRを使った損切りラインの設定方法【具体的な計算式】」
3. 「日本株の単元株とは？100株単位で買えない銘柄はどう扱う？」
4. 「移動平均線（MA25）の乖離率で何がわかるか」
5. 「52週高値・安値レンジで割安・割高を判断する方法」
6. 「パニック相場（VIX30超え）のとき個人投資家はどうすべきか」
7. 「AIに翌日の売買計画を立ててもらった結果【実験レポート】」（Nightly Edgeの宣伝を兼ねる）

### 記事ごとの作業フロー
1. Claude に「○○をテーマに note の有料記事として1,500字で書いて」と指示
2. 人間が内容の正確性を確認・推敲（30分）
3. noteに投稿（300円設定）
4. X で告知ツイート（自動化可能）

### 収益目安
- 1記事あたり月10〜30購入 × 300円 = 3,000〜9,000円
- 10記事蓄積後: 月30,000〜90,000円のストック型収益

## 自動化の全体像（最終形）

```
毎朝8時（GitHub Actions）
  └─ yfinance でVIX・相場データ取得
  └─ Claude でツイート生成
  └─ X に自動投稿

毎週日曜（GitHub Actions）
  └─ yfinance で週次データ取得
  └─ Claude でニュースレター原稿生成
  └─ Markdownファイルを保存
  └─ [人間] 5分確認 → Beehiiv から送信

随時（人間がトリガー）
  └─ Claude で note 記事下書き生成
  └─ [人間] 30分推敲
  └─ note に投稿 → X で告知（これも自動化可）

常時稼働（Railway + Vercel）
  └─ Nightly Edge 本体
  └─ Stripe で課金処理
```

## 優先する理由（重要度ランキング）

1. **X自動投稿** — 工数2〜3日で構築完了、その後ゼロコスト。Nightly Edge への最も効率的な集客手段。
2. **note有料記事** — 一度書けばストック型収益。Claude下書きで人間工数を最小化。
3. **ニュースレター** — 直接収益は後でいいが、読者リストは資産。早めに始めるほど良い。
4. **Nightly Edge機能拡充** — retention向上 → LTV向上。バックテスト表示が最優先。

## 各ステップの移行基準（Go/No-go）

### X → Beehiiv ニュースレター開始
**Go条件（どれか1つ満たせば開始）**
- Xフォロワー 100人超
- 投稿を30本以上続けて、いいね・返信が安定してついてきた
- 「もっと詳しく読みたい」系のコメントが複数来た

**なぜこのタイミングか**: フォロワーが0の状態でニュースレターを始めても読者が集まらない。Xで「読みたい」と思わせてから誘導するのが自然な流れ。

**工数**: アカウント作成30分 + 第1号の確認5分/週。始めるコストはほぼゼロ。

---

### Beehiiv → note 有料記事開始
**Go条件（どれか1つ満たせば開始）**
- ニュースレター読者 50人超
- または X フォロワー 200人超（読者リストと並行でOK）
- 「〇〇についてもっと詳しく教えて」という反応が複数来た

**なぜこのタイミングか**: note は既存ファンへの深堀り販売。フォロワーゼロで出しても誰にも届かない。一方、200人いれば初動10〜20購入は現実的。

**撤退基準**: 記事3本出して合計購入5件未満なら、テーマか価格か認知が足りない。テーマを変えて再挑戦。

---

### X/note → Nightly Edge 機能拡充に投資
**Go条件（両方満たすこと）**
- MAU（月間アクティブユーザー）50人超
- 月次収益 $100 超（集客が機能している証拠）

**なぜこのタイミングか**: 機能を良くしても見てもらえなければ意味がない。集客ができてから retention に投資する順番が正しい。

**優先機能**:
1. バックテスト（過去提案の勝率表示）→ 信頼性UP → コンバージョンUP
2. 週次サマリーメール → 解約率を下げる
3. 韓国語・中国語追加 → アジア圏拡大

---

### Nightly Edge → Product Hunt ローンチ
**Go条件（全部満たすこと）**
- MAU 100人超（実績として語れる）
- 英語LP完成（海外ユーザーが迷わない）
- バックテスト機能実装済み（デモで見せられる）
- 英語ツイートを週2本以上投稿して英語圏の反応を確認済み

**なぜ急がないか**: Product Hunt は1度しか本格ローンチできない。「使える実績」がない状態で出ても埋もれるだけ。弾を温存する。
